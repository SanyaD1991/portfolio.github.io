# Portfolio Builder

Небольшой конструктор портфолио с авторизацией, несколькими подборками, разделами, проектами, достижениями и загрузкой изображений.

## Что уже есть

- вход администратора по email и паролю;
- несколько независимых портфолио;
- публикация/черновик;
- гибкие разделы: текст, проекты, достижения;
- карточки проектов с галереей и тегами;
- настройка цветов каждого портфолио;
- загрузка изображений в Supabase Storage;
- адаптивная публичная часть;
- защита данных через Row Level Security.

## 1. Создайте Supabase-проект

1. Откройте Supabase и создайте проект.
2. В `SQL Editor` вставьте содержимое `supabase/schema.sql` и выполните.
3. В `Authentication → Users` создайте своего пользователя с email и паролем.
4. В `Project Settings → API` скопируйте Project URL и anon public key.

## 2. Настройте проект локально

```bash
cp .env.example .env
```

Заполните `.env`:

```env
VITE_SUPABASE_URL=https://YOUR_PROJECT.supabase.co
VITE_SUPABASE_ANON_KEY=YOUR_ANON_KEY
```

Установка и запуск:

```bash
npm install
npm run dev
```

Редактор будет доступен по адресу `/admin`, публичные портфолио — на главной странице.

## 3. Размещение на Cloudflare Pages

- Build command: `npm run build`
- Build output directory: `dist`
- Добавьте переменные `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY` в настройках проекта.
- Файл `public/_redirects` уже настроен для маршрутов React.

## Что логично добавить следующим этапом

- визуальное перетаскивание разделов;

- кадрирование изображений;
- SEO-поля и Open Graph;

- резервное копирование и экспорт JSON.

## Если npm пытается открыть внутренний registry

Проект настроен на официальный публичный registry npm через файл `.npmrc`.
Если установка ранее прервалась, удалите `node_modules` и повторите `npm install`.
